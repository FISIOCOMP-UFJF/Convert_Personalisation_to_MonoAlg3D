# Eikonal-to-MonoAlg3D Converter

Last updated on: 17/07/2026

## Overview

This tool converts the output of the **Cardiac_Personalisation_Brazil** reduced-order Eikonal personalisation framework into input files compatible with the **MonoAlg3D** cardiac electrophysiology simulator.

The program performs two main tasks:

1. **Transforms inferred Purkinje root nodes** from the personalisation framework into the coordinate system expected by MonoAlg3D.
2. **Calibrates inferred conduction velocities** into monodomain conductivities using the automatic tuning strategy proposed by Costa *et al.* (2013).

The generated files can be directly used to initialize a biophysically detailed monodomain simulation in MonoAlg3D.

---

## Features

* Converts Purkinje root node coordinates from **centimeters (cm)** to **micrometers (µm)**.
* Translates coordinates to the origin using the biventricular mesh bounds.
* Preserves Local Activation Time (LAT) information.
* Calibrates conduction velocities into electrical conductivities.
* Automatically generates MonoAlg3D stimulus configuration sections.
* Automatically generates MonoAlg3D conductivity parameters.

---

## Workflow

```text
Reduced-order Personalisation
        │
        ├── Biventricular LAT mesh (.vtu)
        ├── Root nodes with LAT (.vtk)
        └── Inferred conduction velocities
                 │
                 ▼
      Eikonal-to-MonoAlg3D Converter
                 │
                 ├── Transformed root nodes (.vtk)
                 ├── MonoAlg3D stimulus file
                 └── MonoAlg3D conductivity file
```

## Inputs

The program expects four command-line arguments:

```bash
./bin/Converter \
    <input_biv_lat_file> \
    <input_root_nodes_file> \
    <input_inferred_cvs_file> \
    <output_root_nodes_file>
```

### 1. Biventricular LAT Mesh

VTK UnstructuredGrid (`.vtu`) generated by the personalisation pipeline.

Used to obtain the geometric bounds of the ventricular mesh.

### 2. Root Nodes

VTK PolyData (`.vtk`) containing:

* Purkinje root node locations
* Point data array named `LAT`

### 3. Inferred Conduction Velocities

Text file containing the conduction velocities inferred during personalisation.

Typical parameters include:

* fibre_speed
* sheet_speed
* normal_speed
* endo_dense_speed
* endo_sparse_speed
* purkinje_speed
* fibrosis_speed

### 4. Output Root Nodes

Destination filename for the transformed PolyData.

---

## Outputs

The converter produces three files.

### 1. Transformed Root Nodes

The output VTK PolyData contains:

* coordinates translated to the mesh origin
* coordinates converted from **cm** to **µm**
* original LAT values preserved

---

### 2. MonoAlg3D Conductivity File

```
outputs/monoalg3d_conductivities.txt
```

Example:

```ini
sigma_l = ...
sigma_t = ...
sigma_n = ...

sigma_l_dense = ...
sigma_t_dense = ...
sigma_n_dense = ...

sigma_l_sparse = ...
sigma_t_sparse = ...
sigma_n_sparse = ...

sigma_purkinje = ...

sigma_fibrosis = ...
```

These values are obtained after calibrating the inferred conduction velocities.

---

### 3. MonoAlg3D Stimulus File

```
outputs/monoalg3d_stimulus_section.txt
```

One stimulation section is generated for every inferred Purkinje root node.

Example:

```ini
[stim_1]
start = 12.30
duration = 2.0
current = 1.0
center_x = ...
center_y = ...
center_z = ...
radius = 500.0
main_function = stim_sphere
```

## Coordinate Transformation

The converter applies the following transformation to every root node:

1. Translate the geometry so that the ventricular mesh minimum coordinate becomes the origin.
2. Scale coordinates from

```
centimeters → micrometers
```

using

```
1 cm = 10,000 µm
```

This matches the coordinate convention adopted by MonoAlg3D.

---

## Conduction Velocity Calibration

The inferred conduction velocities are converted into monodomain conductivities through the calibration routine implemented in `tunecv.cpp`.

This follows the methodology described in:

> Costa, C. M., Hoetzl, E., Rocha, B. M., Prassl, A. J., & Plank, G.
> *Automatic parameterization strategy for cardiac electrophysiology simulations.*
> Computing in Cardiology, 2013.

---

## Dependencies

The project requires:

* C++11 or newer
* VTK
* Calibration library (`tunecv.h`)

VTK classes used include:

* `vtkXMLUnstructuredGridReader`
* `vtkGenericDataObjectReader`
* `vtkTransform`
* `vtkTransformPolyDataFilter`
* `vtkPolyDataWriter`
